**The Zen of Odyn** is a set of principles that drive which features get added and why certain decisions are made the way they are. When something about Odyn seems unintuitive, the answer is usually here.

1. Odyn does one thing. It does not accumulate.
2. Scoped to the project, not the machine.
3. The user is always in control. Odyn never decides on their behalf.
4. Transparency is a feature, not a limitation.
5. Friction that belongs elsewhere stays elsewhere.
6. Explicit over automatic.
7. The lockfile is the truth.
8. If it can't be undone by hand, Odyn shouldn't do it automatically.
