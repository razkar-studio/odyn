Odyn manages your Odin project's dependencies by cloning Git repositories into `odyn_deps/` and pinning exact commits in a file called `Odyn.lock`. You could replicate it with Git and a man with a spreadsheet. 

Literally a "glorified git clone + lockfile," just your dependencies, where you put them, at exactly the commit you asked for.

## Quick Start

```sh
# Create a new Odin project
odyn init myproject
cd myproject

# Add a dependency
odyn get odin-community/math

# Import it in your code
# import "deps:math"

# Reproduce the exact same state on any machine
odyn sync
```

`odyn init` scaffolds your project and sets up `ols.json` with the `deps:` collection. `odyn get` clones and pins. `odyn sync` makes everything match the lockfile.

## Commands

Commands marked with a checkmark (:white_check_mark:) is complete. The rest is a work in progress.

| Command | Description |
|---|---|
| :white_check_mark: `odyn init <name>` | Scaffold a new Odin project with `src/`, `odyn_deps/`, `ols.json`, and an empty `Odyn.lock` |
| :white_check_mark: `odyn get <source> [name]` | Clone a dependency and pin its commit in `Odyn.lock`. *(COMING SOON) Accepts `user/repo` shorthand (GitHub default) or a full URL* |
| :white_check_mark: `odyn sync` | Make `odyn_deps/` match `Odyn.lock` exactly. Re-clones missing deps, errors on modified ones |
| `odyn status` | Show the state of every dependency — ok, missing, or modified |
| `odyn update <name>` | Fetch the latest commit for a dependency and re-pin it in `Odyn.lock` |
| :white_check_mark: `odyn remove <name>` | Delete the dependency folder and remove its entry from `Odyn.lock` |
| `odyn update-self` | Update Odyn itself *(coming soon)* |

### `odyn init` flags :white_check_mark:

| Flag | Description |
|---|---|
| `--license <type>` | License to generate. Defaults to `mit`. Also accepts `apache` |
| `--with-readme` | Add a `README.md` stub |
| `--no-src` | Skip creating the `src/` directory |

### `odyn get` flags

| Flag | Description |
|---|---|
| `--codeberg` | Resolve `user/repo` shorthand against Codeberg instead of GitHub |

## The Lockfile

Every time you run `odyn get`, Odyn writes an entry to `Odyn.lock`:

```toml
# This file is automatically @generated by Odyn.
# Do not edit this file manually unless you know what you're doing.

[[dep]]
name = "math"
source = "https://github.com/odin-community/math"
commit = "a1b2c3d4e5f6a1b2c3d4e5f6a1b2c3d4e5f6a1b2"
```

Commit `Odyn.lock` to your repository. That's what makes `odyn sync` work, anyone who clones your project runs one command and gets the exact same dependencies at the exact same commits. No surprises.

You can gitignore `odyn_deps/` or commit it too. Your call. The lockfile is what matters.

## Importing Dependencies

Odyn sets up your `ols.json` to register `odyn_deps/` as a collection called `deps`. So anywhere in your project:

```odin
import "deps:math"
```

No relative paths. No `../../odyn_deps/math`. Just `deps:name`, from any file, always.

When building, pass the collection flag to the compiler:

```sh
odin run src -collection:deps=odyn_deps
```

## Building from Source

You'll need Rust 1.85.0 or newer (Odyn uses Rust edition 2024).

```sh
git clone https://codeberg.org/razkar/odyn.git
cd odyn
cargo build --release
```

The binary lands at `target/release/odyn`. Put it somewhere on your `PATH` and you're set.

> **Note:** Prebuilt binaries and an install script are coming soon. For now, building from source is the way to go.
